# 本地化风格指南

## 目标

本仓库的中文化目标是：在不破坏现有路径、资源名、链接、锚点和代码示例的前提下，将面向中文读者的可见文本翻译为自然、统一、可维护的简体中文。

## 基本规则

- 保留路径、文件名、URL、命令、代码块、内联代码原样。
- 保留外部产品名、协议名、参数名、模型名的英文原词；必要时首次出现可加中文解释。
- 图片 `alt`、HTML `title`、导航文案、SVG 文本属于可翻译内容。
- 不为追求逐字直译而牺牲可读性，优先自然中文表达。
- 同一术语在全仓保持一致，除非上下文明确要求不同译法。

## 标题规则

- 标题优先使用简洁中文，不保留生硬的英文直译腔。
- 如标题已被站内锚点引用，保留旧锚点别名，不回避中文标题。
- 列表标题、表格列名、分节标题统一使用中文。

## 术语保留

- `Claude Code`、`Codex`、`MCP`、`SDK`、`CLI`、`API`、`OAuth`、`GitHub`、`Slack` 等品牌或缩写保留英文。
- `Subagents`、`Commands`、`Skills`、`Hooks`、`Memory`、`Settings` 等高频概念按术语表统一翻译，首次出现可中英并列。
- `agentic engineering` 统一译为“智能体工程”，首次出现可写作“智能体工程（agentic engineering）”。

## 不翻译项

- 代码块
- 内联代码
- 命令行参数
- 路径和文件名
- JSON 键名
- HTML/CSS/JS 标识符
- 外链目标 URL

## 图片与资源

- SVG 内部的纯文本节点要翻译。
- SVG 若需重排，以版式完整性优先，不强行塞入逐字等长文本。
- HTML 页面文案必须和机读 manifest 对齐。

## 语气

- 说明文以清晰、直接、工程化表达为主。
- 教程文案偏引导式，但避免口语化过度。
- 不夸张，不使用营销腔，不堆叠感叹号。
