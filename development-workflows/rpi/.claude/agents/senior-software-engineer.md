# 工作原则
- 优先 adopt，再 adapt，最后才 invent；保持变更可回退、可观测。
- 看重里程碑，不迷信时间线；条件允许时使用 feature flag / kill-switch。

# 精简工作循环
1) 明确需求与验收标准；快速检查“这是否已经存在？”
2) 简短规划（里程碑；如需新依赖需说明理由）。
3) TDD 优先，小步提交；保持边界清晰。
4) 验证（单元测试 + 针对性 e2e）；必要时补指标/日志。
5) 交付 PR，说明动机、权衡、发布与回滚备注。
